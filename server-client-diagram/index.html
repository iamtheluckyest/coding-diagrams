Request-Response Cycle

Client
  •   
  •   
  •   
Definition
Sends requests, receives responses
Examples

The browser, which sends a GET request any time it goes to a new URL and displays the response.


AJAX, which sends requests (i.e. when triggered by user input) and stores the response as data in a variable.

Clues you're coding the client-side
  • You are writing HTML files
  • You are using jQuery
  • You are sending requests with AJAX
Request
  •   
  •   
  •   
Definition
Data sent by the client to the server. Holds information in head and body.
Head
Contains information about the request itself, such as the URL, the method (GET, POST, etc), or content type.
Body
Contains any other data that the client is passing to the sever. In AJAX, this is the information assigned to the “data” key.
Response
  •   
  •   
  •   
Definition
Data sent by the client to the server. Holds information in head and body.
Head
Contains information about the response itself, such as the status, or content type.
Body
Contains any other data that the server is passing to the client. In AJAX, this received in the .done callback.
Server
  •   
  •   
  •   
Definition
Receives requests, sends responses
Examples

Code logic that parses the URL, method, and other data from a request, and sends back data.


Software like Express/Node or Apache that enables such code to function.

Clues you're coding the server-side
(in Node.js with Express)
  • You are using require()
  • You are using express()
  • You are using routers (i.e. app.get(), router.get())
